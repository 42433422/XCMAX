# XCAGI v10.0 快速启动指南

> 版本：10.0.0 | 更新日期：2026-06-07

XCAGI v10.0 支持**桌面版**和 **Web 版**两种运行模式。

---

## ⚡ 快速启动（桌面版）

### 方式 1：一键启动脚本（推荐）

```cmd
# Windows 桌面版
双击 start-dev.bat
或
双击 quick-start.bat
```

### 方式 2：手动启动

```bash
# 1. 进入项目目录
cd e:\FHD

# 2. 启用桌面模式
set XCAGI_DESKTOP_MODE=1

# 3. 启动服务（默认端口 5000）
python run.py

# 4. 访问应用
# 前端: http://localhost:5173（开发服务器）
# API:  http://localhost:5000
```

---

## ⚡ 快速启动（Web 版 / Docker）

### 方式 1：使用部署脚本

```cmd
# Windows
deploy.bat
# 选择选项 1（生产环境部署）
```

### 方式 2：Docker Compose

```bash
# 1. 进入项目目录
cd e:\FHD

# 2. 复制环境变量文件
copy .env.example .env

# 3. 编辑 .env，配置必要的密钥和 API Key

# 4. 构建并启动
docker-compose up -d --build

# 5. 查看状态
docker-compose ps

# 6. 查看日志
docker-compose logs -f
```

## 📋 常用命令速查

### 启动服务

```bash
# 桌面版（开发模式）
python run.py

# Web 版（Docker）
docker-compose up -d

# 开发环境（Docker）
docker-compose -f docker-compose.dev.yml up -d
```

### 查看状态

```bash
# Docker 容器状态
docker-compose ps

# 健康检查
curl http://localhost:5000/api/health
```

### 查看日志

```bash
# Docker 日志
docker-compose logs -f
docker-compose logs -f backend
docker-compose logs -f frontend

# 本地日志
# 查看 logs/ 目录
```

### 停止服务

```bash
# Docker
docker-compose down

# 停止并清理数据（危险！）
docker-compose down -v
```

## ❓ 常见问题

### Q1: 端口被占用

**解决**：
```bash
# 查看占用端口的进程（Windows）
netstat -ano | findstr :5000
taskkill /PID <PID> /F

# 或修改启动脚本中的端口
```

### Q2: 服务启动后立即停止

**解决**：
```bash
# 查看详细日志
python run.py
# 检查 .env 配置
# 确认 Python 版本 >= 3.10
python --version
```

### Q3: 桌面模式不工作

**解决**：
```bash
# 确认环境变量已设置
echo %XCAGI_DESKTOP_MODE%
# 应该输出 1

# 检查是否使用 SQLite（桌面模式应使用 SQLite）
```

### Q4: 前端无法访问

**解决**：
```bash
# 开发模式：确认前端开发服务器已启动
cd frontend
npm run dev

# Docker 模式：检查容器状态
docker-compose ps frontend
docker-compose logs frontend
```

### Web 版（Docker）
- **前端**: http://localhost
- **API**: http://localhost:5000
- **API 文档**: http://localhost:5000/docs

## 🔄 升级到 v7.0

如果从旧版升级：

```bash
# 1. 备份数据
# Docker 模式
docker-compose down
docker run --rm -v xcagi-data:/data -v $(pwd)/backup:/backup alpine \
  tar czf /backup/xcagi-data-backup.tar.gz /data

# 2. 更新代码
git pull origin master

# 3. 更新 .env 配置
# 添加 XCAGI_DESKTOP_MODE=1（如需桌面版）

# 4. 重新构建
docker-compose build --no-cache

# 5. 启动服务
docker-compose up -d

# 6. 运行数据库迁移
docker exec xcagi-backend alembic upgrade head
```
