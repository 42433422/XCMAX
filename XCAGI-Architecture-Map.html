典型请求：用户发送聊天消息 → 出货单生成

1
HTTP 请求到达
POST /api/chat → fastapi_app.py → middleware链
XCAGI/run.py 启动 uvicorn → app/fastapi_app.py:get_fastapi_app() 工厂创建应用
请求依次经过: CSRFMiddleware → SecurityHeadersMiddleware → XSSSanitizerMiddleware → ModContextMiddleware → LanCidrGuard → LanLicenseGuard
最终路由到 app/fastapi_routes/ai_assistant.py
2
路由层 → 应用服务
ai_assistant.py → ai_chat_app_service.py
路由层薄封装，通过 bootstrap.py 或 di/registry.py 获取应用服务实例
@login_required 装饰器验证身份 → 调用 AiChatAppService.process_chat()
3
应用服务编排用例
ai_chat_app_service.py → workflow/ → facades/
process_chat() 核心流程（~2800行）：
1. 确认流程检查 → 2. 动态工作流分发 → 3. AI服务调用 → 4. 响应构建
Pro模式: workflow/planner.py LLM规划 → workflow/engine.py 执行
Normal模式: 槽位路由 → 即时工具执行
工具调用通过 facades/ 门面（如 ShipmentFacade、PrintFacade）
4
领域层执行业务规则
domain/shipment/ → domain/services/ → domain/neuro/
domain/shipment/aggregates.py Shipment聚合根 + ShipmentItem实体
domain/shipment/shipment_product_parser.py 四级产品匹配（型号精确→名称包含）
domain/services/unified_intent_recognizer.py 四引擎级联意图识别
domain/neuro/processors/coordinator.py 三级处理器协调（Reflex→Subconscious→Conscious）
5
基础设施层执行 I/O
infrastructure/persistence/ → infrastructure/llm/ → infrastructure/printing/
infrastructure/persistence/shipment_record_command_impl.py 写入出货记录
infrastructure/persistence/shipment_record_store_impl.py 持久化存储
infrastructure/llm/client.py 调用DeepSeek等LLM
infrastructure/printing/printer_adapter.py 打印适配器
所有基础设施实现 application/ports/ 中定义的Port接口
6
NeuroBus 事件发布 + 数据落库
neuro_bus/domains/shipment_domain.py → db/models/shipment.py → PostgreSQL
出货创建后发布 shipment.created 事件到NeuroBus
库存域订阅该事件 → 自动扣减库存（inventory_domain_handlers.py）
审批服务检查是否需要审批（workflow/approval_service.py）
数据通过 db/session.py 的 SQLAlchemy Session 写入 PostgreSQL
NEUROBUS 神经总线
app/neuro_bus/bus.py (~700行) · PriorityEventQueue · asyncio异步处理循环
去重 Deduplicator
限流 RateLimiter
熔断 CircuitBreaker
追踪 Tracer
SLA日志
保命通道 Lifeline
死信队列 DLQ
重试 RetryHandler

代码依赖方向规则（DDD分层约束）

Route → Application
fastapi_routes/ → application/
路由层只调用应用服务，不直接操作领域或基础设施
通过 bootstrap.py 或 di/registry.py 获取服务实例
Application → Domain + Port
application/ → domain/ + application/ports/
应用服务调用领域规则，通过Port抽象访问基础设施
application → infrastructure（禁止直接依赖）
Domain → 无外部依赖
domain/ 不依赖任何外部模块
纯领域规则、实体、值对象
domain → infrastructure（禁止）
domain → application（禁止反向依赖）
Infrastructure → Port + Domain
infrastructure/ 实现 application/ports/ 定义的接口
仓储实现使用 db/models/ ORM模型
适配器封装外部系统（LLM/OCR/打印/支付）
NeuroBus 横切
neuro_bus/ 贯穿所有层，但不被领域层直接依赖
应用服务通过 instrument_*() 插桩
神经域通过 emit() 发布事件，处理器在 *_handlers.py
Mod SDK 横切
mod_sdk/ 提供Mod开发接口
*_compat.py 桥接宿主与Mod的API/页面
Mod通过 hooks 订阅宿主事件
Mod通过 comms 与其他Mod点对点通信
硬约束（提交前自检）
1. 服务端代码只许写到 app/
2. 前端代码只许写到 frontend/
3. HTTP入口只用 XCAGI/run.py（端口5000）
4. 禁止新增 backend/ 目录（已删除）
5. 禁止往 static/ templates/ 写源码
装配根
app/bootstrap.py 是 Composition Root
使用 @lru_cache 装配应用服务与基础设施
app/di/registry.py ServiceContainer 管理重型单例
所有接线集中在这两处，不在路由里 new 对象
