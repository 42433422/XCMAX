| 文件 | 说明 |
|------|------|
| `../template.docx` | **1_完善版_字段模板.docx** — 系统填充母版（`{{field_key}}`） |

## 1. 法大大控制台

### 1.1 跨环境导入模板（推荐，已生成分享链接）

在 **企业管理后台 → 模板管理 → 导入模板** 弹窗中填写：

| 项 | 值 |
|----|-----|
| 模板链接 | `9544cb669ca341cc910a8978e7e952d9` |
| 密码 | `4iup` |

本机备份见同目录 `import_template.local.env`（已加入 `.gitignore`，勿提交 Git）。

导入成功后：

1. 打开该签署模板，确认参与方 **actorId** 含 `甲方`（客户签署）。
2. 复制 **签署模板 ID** → 写入 `FHD/.env` 的 `FADADA_SIGN_TEMPLATE_ID`。
3. 若参与方 ID 与代码不一致，设置 `FADADA_SIGN_ACTOR_ID` 为模板里客户方的 actorId。

### 1.2 本地上传（备选）

1. [法大大开放平台](https://dev.fadada.com/) 创建应用 → `FADADA_APP_ID` / `FADADA_APP_SECRET`
2. 企业认证（成都修茈）→ `FADADA_OPEN_CORP_ID`
3. **模板管理** → 上传 **`法大大上传_template.docx`**（约 46KB，含完整条款与 98 处占位符）
4. 配置 **签署模板**：
   - **发起方**：企业（乙方 openCorpId）
   - **个人签署方 actorId**：`甲方`（客户）→ 与 `FADADA_SIGN_ACTOR_ID=甲方` 一致
5. 拖好签章位置 → 发布 → 复制 **签署模板 ID** → `FADADA_SIGN_TEMPLATE_ID`
6. 回调：`https://你的域名/api/contract-lifecycle/esign/webhook`

## 2. FHD 环境变量

复制 [`config/fadada.env.example`](../../../config/fadada.env.example) 到 `FHD/.env`：

```bash
ESIGN_PROVIDER=fadada
FADADA_APP_ID=
FADADA_APP_SECRET=
FADADA_OPEN_CORP_ID=
FADADA_SIGN_TEMPLATE_ID=
FADADA_SERVER_URL=https://api.fadada.com/api/v5
FADADA_CALLBACK_URL=https://你的域名/api/contract-lifecycle/esign/webhook
FADADA_SIGN_ACTOR_ID=甲方
```

## 3. XCAGI 使用

- **生成 Word 合同**：内部客服 → 合同区 → 按 `fill_config.json` 填充 `template.docx`
- **电子签**：财务统计 / 客服 → 发起电子签 → 客户打开法大大 `sign_url` → 回调后合同 **effective**

## 5. 说明

- 占位符语法：`{{party_a_name}}` 等，与 `fill_config.json` 中 `fields.*.placeholder` 一致。
- 法大大若要求「填写控件」，可按 `template_fields_manifest.json` 的 key 命名。
- 完整条款以 `template.docx` 为准；`sample_party_b_prefilled.pdf` 仅作乙方预填参考，不必上传法大大。
