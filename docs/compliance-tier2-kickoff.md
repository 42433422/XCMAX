| 项 | 内容 |

## 2. 机构接洽

### 2.1 短名单（建议 3 家）

| # | 动作 | 完成 |
|---|------|------|
| 1 | 从**公安部备案的测评机构名录**筛 3 家（同城或熟悉本省备案流程者优先） | ⬜ |
| 2 | 发 **RFP 简表**（见 §2.2）索取：报价区间、周期、是否含整改咨询、驻场次数 | ⬜ |
| 3 | 商务 / 法务 / 技术三方 **30min 答疑会**（每家 1 次） | ⬜ |
| 4 | 比价表 + 推荐机构 **内部签字**（负责人 + 法务） | ⬜ |

### 2.2 RFP 简表（可复制发邮件）

```text
【等保二级】询价要点 — 成都修茈 / XCMAX FHD SaaS
1. 定级建议：二级；系统描述见附件一页架构图
2. 部署：________（云/机房/地域）
3. 期望时间表：合同签署后 ___ 周内出测评计划；目标拿证窗口 ___（对齐 M6-W1）
4. 报价：测评费 + 整改辅导（人天）+ 复测次数是否含内
5. 交付物：测评报告、整改清单模板、备案辅导范围
6. 保密与数据：现场测评数据范围、脱敏要求
```

## 3. 合同

### 3.1 签约前检查

| # | 条款 | 完成 |
|---|------|------|
| 1 | **服务范围**：二级测评 + 报告 + 约定次数复测；整改咨询是否另计费写清 | ⬜ |
| 2 | **系统边界**：URL / 子系统清单附件；与 MODstore、支付链路是否纳入 | ⬜ |
| 3 | **甲方配合义务**：提供拓扑、账号、访谈窗口；**不含**真实生产密钥出库 | ⬜ |
| 4 | **周期与里程碑**：与 §4 时间表一致；逾期责任 | ⬜ |
| 5 | **保密 / 数据安全**：测评数据销毁、人员保密协议 | ⬜ |
| 6 | **付款**：首付 / 报告款 / 尾款节点 | ⬜ |
| 7 | **法务**：主体、发票、争议管辖（与现有采购流程一致） | ⬜ |

### 3.2 签约后 48h 内

| # | 动作 | 完成 |
|---|------|------|
| 1 | 合同 PDF（脱敏版可入仓）→ `evidence/compliance-tier2/01-contract-redacted.pdf` | ⬜ |
| 2 | 指定**甲方项目经理**（技术 + 商务各 1 人）与机构对接群 | ⬜ |
| 3 | 更新 [`M1-kickoff-checklist.md`](M1-kickoff-checklist.md) M3-W4 行 → ✅（仅当合同 + 时间表两项齐备） | ⬜ |

## 4. 时间表（书面里程碑）

> 下列为**模板**；签约后以机构《项目实施计划》为准，但须覆盖各阶段**起止周**与**甲方交付物**。

| 阶段 | 典型周期 | 甲方交付 | 机构交付 | 计划起 | 计划止 |
|------|----------|----------|----------|--------|--------|
| T0 启动 | 第 1 周 | 合同、架构一页图、资产清单初稿 | 项目计划书 | | |
| T1 定级与备案辅导 | 2–4 周 | 定级报告素材、网络拓扑 | 定级建议、备案材料清单 | | |
| T2 差距分析 / 整改 | 4–8 周 | 按《整改清单》改配置、制度、日志留存 | 差距表、整改复核 | | |
| T3 现场测评 | 1–2 周 | 访谈、演示账号、机房/云配合 | 现场记录 | | |
| T4 报告与复测 | 2–4 周 | 复测项关闭证明 | 测评报告、合格结论 | | |
| **M6-W1 拿证** | 路径图目标 | 备案证明归档 | 公安机关备案回执（以当地流程为准） | | |

**M3-W4 完成定义**：上表 **T0** 行「计划起/止」已填，且与合同附件或机构计划书 **一致**（PDF 或双方邮件确认截图）。

### 4.1 与 M1 其他任务并行建议

| 并行项 | 说明 |
|--------|------|
| M5-W1 审计日志 | 整改期常需操作/登录审计；可提前排期开发 |
| M6-W2 脱敏加密 | 与等保控制项重叠，避免测评后二次大改 |
| staging SLO | **无硬依赖**；但月报 / 容量数据可辅助「安全运维」制度佐证 |
