# 每周执行检查清单（`specs/weekly/`）

> **用途**：把 [`对标头部SaaS-步骤路径图.md`](../对标头部SaaS-步骤路径图.md) §6 的每周模板落成可归档文件，供执行者 / review 使用。  
> **不替代**：季度战术仍以 [`plan-2026-06.md`](../plan-2026-06.md) + [`plan-2026-06-checklist.md`](../plan-2026-06-checklist.md) 打勾；阻塞以 [`BLOCKERS.md`](../BLOCKERS.md) 为准；自证数字以 [`FHD/docs/CLAIMED_VS_ACTUAL.md`](../FHD/docs/CLAIMED_VS_ACTUAL.md) 为准。

## 命名

- 文件：`YYYY-Www.md`（ISO 周，例如 `2026-W23.md`）
- 一周一份；周末或周一补「完成 / 证据」
- **提交位置**：在 **`XCMAX/` 仓根** commit / push（[`42433422/XCMAX`](https://github.com/42433422/XCMAX)）；`specs/weekly/`、`FHD/`、`Makefile` 等同仓提交。CI 见 [`docs/CI_SSOT.md`](../../docs/CI_SSOT.md)。

## 模板来源

复制 [`对标头部SaaS-步骤路径图.md`](../对标头部SaaS-步骤路径图.md) §6，或从上一周文件 fork。每份须包含：

1. **本周目标**（≤3 条）
2. **证据**（截图 / 链接 / 文档路径占位）
3. **与 BLOCKERS / CLAIMED 交叉核对**（见各周文件底部固定段）

## CI

本目录 **不** 接入 CI；人工 review 即可。
