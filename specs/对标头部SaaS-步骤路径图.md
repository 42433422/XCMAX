## 1. 飞书 / 钉钉 必备能力矩阵（共 12 项）

把头部 SaaS 拆成 12 个能力域，**逐项对账**。每项打分（XCMAX / 飞书钉钉）+ 差距原因。

### 1.1 基础通信（IM / 音视频 / 邮件）
- **XCMAX**：❌ 无 IM、无音视频。**WXCC 小程序**有微信生态间接通信。
- **飞书钉钉**：✅ 完整自研 IM（消息、群、已读）、自研音视频、邮件日历一体化。
- **差距**：🔴🔴 2–3 年。**短期不自研**，集成第三方（环信 / 融云 / 声网）即可。

### 1.2 协作（文档 / 表格 / 云盘）
- **XCMAX**：❌ 无在线文档；Excel 模板是业务模板，非实时协作。
- **飞书钉钉**：✅ 飞书文档（类 Notion）/钉钉文档（类 Office）/云盘。
- **差距**：🔴 大。**集成腾讯文档/飞书文档 SDK** 是务实选项。

### 1.3 审批 / 工作流
- **XCMAX**：✅ 已有 `approval.py`（1067 行）+ `app/application/workflow/` 引擎。
- **飞书钉钉**：✅ 自研审批引擎 + 低代码。
- **差距**：⚠️ 中。**引擎有，但缺低代码搭建**（这是飞书钉钉杀手锏）。

### 1.4 任务 / 项目
- **XCMAX**：❌ 无。
- **飞书钉钉**：✅ 任务中心 + 项目空间。
- **差距**：🔴 大。**可缓**（与 ERP 业务弱相关）。

### 1.5 智能（AI 助手）
- **XCMAX**：⚠️ AI 聊天/审单/意图识别/Excel 向量，但**仍以 chat 为主**。
- **飞书钉钉**：✅ 智能纪要、智能搜索、AI 员工、Agent 市场。
- **差距**：⚠️ 中。**从 chat → Agent 是质变**。

### 1.6 应用商店 / 生态
- **XCMAX**：⚠️ Mod 商店雏形（[`MODstore/`](file:///Users/a4243342/Desktop/XCMAX/FHD/MODstore/)），上架流程 + 计费框架已有，**真实商家 0**。
- **飞书钉钉**：✅ 千 + 应用、年 GMV 亿级。
- **差距**：🔴 大。**真实商家跑通是分水岭**。

### 1.7 开放平台（API / SDK / 事件 / Webhook）
- **XCMAX**：✅ Mod SDK 2.0 / OpenAPI 已有。
- **飞书钉钉**：✅ 完整开发者门户 + 沙箱 + 计费。
- **差距**：⚠️ 中。**门户/沙箱/计费是硬骨头**。

### 1.8 企业服务（多租户 / 权限 / 审计 / SSO）
- **XCMAX**：⚠️ 多租户/权限/审计部分有，**SSO（SAML/OIDC）半成品**（[`auth/oidc_provider.py`](file:///Users/a4243342/Desktop/XCMAX/FHD/app/infrastructure/auth/oidc_provider.py) / [`saml_provider.py`](file:///Users/a4243342/Desktop/XCMAX/FHD/app/infrastructure/auth/saml_provider.py) 待验证）。
- **飞书钉钉**：✅ 全套 + 字段级权限。
- **差距**：⚠️ 中。**SSO 必须补**（大客户卡这里）。

### 1.9 合规与安全（等保 / ISO / 加密 / 审计）
- **XCMAX**：❌ 缺等保认证、ISO 27001 缺、密钥管理弱。
- **飞书钉钉**：✅ 等保三级 + ISO 27001 + SOC 2。
- **差距**：🔴 大。**大客户签约硬卡**。

### 1.10 性能 / 规模 / SLA
- **XCMAX**：❌ 0 验证（CLAIMED_VS_ACTUAL 全「未验证」）。
- **飞书钉钉**：✅ 99.99% SLA 公开承诺。
- **差距**：🔴🔴 致命。**补 staging 7 天流量是起点**。

### 1.11 商业化（自助 / 支付 / 阶梯定价 / 续费）
- **XCMAX**：⚠️ Mod 商店支付框架有（[`alipay_package/`](file:///Users/a4243342/Desktop/XCMAX/成都修茈科技有限公司/alipay_package/)），**主产品无自助开通**。

- **差距**：🔴 大。**漏斗最弱的一环**。

### 1.12 客户端体验（移动 / 桌面 / Web / 小程序一致性）
- **XCMAX**：⚠️ Web + 桌面 + 小程序 + Android（实验）。**移动端仍是规划**。
- **飞书钉钉**：✅ 四端体验一致 + 离线 + 弱网。

## 3. 6 阶段步骤路径图（0–36 月）

> **核心思想**：
> - **M0–M1**：补证据、收卫生（不动功能，纯治理）
> - **M1–M3**：自证 + AI 深化（拿截图、跑通真实场景）
> - **M3–M6**：SaaS 化基础（自助/SSO/合规/集成）
> - **M6–M12**：生态 + AI Agent
> - **M12–M24**：规模 + 多端打磨
> - **M24–M36**：音视频 + 国际化（可选）

## 9. 总结

XCMAX 与飞书钉钉差距是**「广度」+「证据」**，不是**「深度」**。
- **深度（ERP+AI 员工）**：✅ 已具备，护城河在
- **广度（IM/音视频/文档/任务）**：❌ 缺，**用集成换时间**
- **证据（SLO/AI 月报/真实商家）**：❌ 致命短板，**M0 立即补**
- **生态（应用商店）**：⚠️ 有雏形，**真实商家是分水岭**

**36 个月可对标飞书钉钉「基础套件」**（不含自研 IM/文档/音视频）。
**核心战略**：做深 AI 员工 + ERP，集成做广协作。
