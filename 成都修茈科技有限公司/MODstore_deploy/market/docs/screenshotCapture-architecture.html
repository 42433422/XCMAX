1 · 公开契约（唯一 import 入口）

screenshotCapture.public.ts 仅 re-export 三类东西：

  • 函数：captureViewport / invalidateCaptureCache / onCaptureMeta
  • 类型：CaptureResult 判别联合（image / text-snapshot / failure）
  • 枚举：CaptureFailureReason、Severity、ScreenshotBackend

实现文件 screenshotCapture.ts 是模块内部，重构时不破坏契约。

2 · 主流程（4 段）

  • 归一化 + 前置检查：defaults → signal 已 abort → document.body 存在
  • 缓存层：key 含 backend|route|scale|quality|viewport|ignore，LRU 上限 5，TTL 30s；forceRetake=true 绕过
  • 后端路由：html2canvas / dom-snapshot，未知 → no_backend
  • 写缓存 + 遥测：成功失败都缓存（避免雪崩），始终 emit meta

3 · 后端 A：html2canvas

  • 动态 import('html2canvas')，首次失败 → render
  • 渲染 → signal.aborted 后置检查 → toDataURL('image/jpeg', quality) → bytes = (len/4)*3 − padding
  • 渲染完立即 canvas.width=0 / height=0 释放 ~31MB
  • 错误分类：cors / csp / memory / timeout / aborted / render
  • 仅 transient（cors/memory/timeout）才按 100/200/400ms 指数退避重试，最多重试 retry 次

4 · 后端 B：dom-snapshot

  • 零依赖兜底，永远不 CORS / CSP / 失败
  • 复用 serializeVisibleDom() 收集：标题 / 路径 / h1-h3 / 可见按钮 / 输入框 placeholder / 表格列
  • JSON 编码 → UTF-8 safe base64 → data:application/json;base64,...
  • kind='text-snapshot'，bytes=json.length

5 · 失败兜底（任何 backend 失败都走这里）

  • _buildFailure 构造 {ok:false, reason, severity, ...}
  • 若 autoTextFallback=true：调用 serializeVisibleDom() 附 textFallback，按 noteMaxLen 截断并标记 noteTruncated / noteOriginalLength
  • severity 映射：aborted→user · transient→transient · 有 fallback→degraded · 无 fallback→critical

6 · 缓存 & 遥测

  • 缓存：模块级 Map，LRU 上限 5，TTL 30s；成功与失败结果都缓存（防雪崩）
  • 失效 API：invalidateCaptureCache(key?) — 不传清空、传 key 精准删
  • 遥测：onCaptureMeta(listener) 注册幂等；listener 抛错被吞掉，不影响 capture 链路
